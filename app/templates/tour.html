Getting started

Overview of the project, it's contents, and how to get started with a simple template.

Why?

Creating mock services saves time and lets the UI developer get started without waiting on backedn APIs.

Take the following sales data example. You get a request for a feature that displays sales data with a pie chart.

Sales Data Example

You want to prototype the feature to make sure everyone is on the same page. You have some choices:

  • 1. You can hardcode the data on the page. This is fine but requires time and your code is useless.
  • 2. You can mock your services, but this can be time consuming as well.
